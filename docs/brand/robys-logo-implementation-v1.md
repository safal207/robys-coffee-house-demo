# Roby's identity normalization v1

Source decision: the owner-provided Roby's identity sheet and the repository identity audit.

## Implemented

- uses path-only primary, medium, compact and mark masters;
- derives favicon, maskable icon and Apple touch icon from the approved organic O;
- statically links the Apple touch icon in Home, Menu, Discover, 404 and owner utility pages, with JavaScript retained only as a progressive fallback;
- uses canonical digital identity tokens: red `#E21B23`, ink `#111111`, and paper `#F5F5F2`;
- aligns the shared UI ruby token, owner utility pages and offline surface to the canonical red;
- replaces visible legacy `R` badges in the map, owner utilities and 404 with approved SVG assets;
- keeps `Fresh Coffee Point` as the structured brand slogan while allowing marketing headlines as a separate copy layer;
- retires `robys-mobile-master-v1.svg`: the compact transparent master is the asset, while pill radius and shadow belong to CSS;
- verifies source assets, static wiring, safe zones, supporting pages, manifest delivery and legacy-token rejection.

## Evidence boundary

This implementation does not claim trademark clearance, consumer recall, storefront-distance readability, physical print durability or revenue impact. One-color/reverse production masters and physical mockup testing remain separate work.
